# MVPs — Canonical Development Path

Este proyecto se desarrolla a través de una secuencia de MVPs canónicos.
Cada MVP define un sistema ejecutable, estable y observable.

No se trabaja fuera de estos MVPs.

Orden obligatorio:
1. MVP_01_PREVIEW
2. MVP_02_CPP_FILTER
3. MVP_03_ANALYZER
4. MVP_04_UDP

Si algo rompe un MVP, no se continúa.
